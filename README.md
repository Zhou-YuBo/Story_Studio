# Story Studio

### 本地优先的长篇故事创作工作台

不是 AI 代写器，也不只是文本编辑器。Story Studio 试图把长篇创作背后的系统整理出来：灵感从哪里来，人物为什么行动，结构如何变化，场景怎样推进，以及写作时哪些约束不能忘。

## 当前版本

Story Studio 当前处于 `v0.1.0-alpha.2` 预览版阶段。

这个版本适合：

- 体验 Story Studio 的整体创作工作流。
- 用本地项目文件试写、试整理故事。
- 反馈工作台设计、界面逻辑和真实创作流程问题。
- 参与一个仍在形成中的开源创作工具。

这个版本暂不适合：

- 作为唯一正式生产环境保存重要项目。
- 期待跨版本项目文件完全兼容。
- 期待云同步、多人协作或 AI 自动写作。
- 期待法律意义上的版权登记、公证或可信时间戳服务。

Alpha 阶段的数据结构和界面细节仍可能变化。升级版本或处理重要项目之前，请保留额外备份。

## Story Studio 是什么

Story Studio 是一个面向小说作者、编剧、游戏叙事设计师、动画导演和其他长篇故事创作者的本地桌面应用。

很多写作软件主要解决“把文字写下来”的问题。Story Studio 更关心的是另一个问题：当一个故事变得足够长、人物足够多、结构足够复杂时，创作者如何继续掌控它？

它把故事拆成几个彼此关联的工作区：

- 灵感和素材从哪里来。
- 世界观、规则、地点、阵营如何组织。
- 人物的欲望、关系和弧光如何互相牵引。
- 幕、序列、场景之间如何形成结构。
- 正文写作时如何看见场景节拍和结构位置。
- 哪些提醒应该在特定工作台或创作范围内出现。
- 最终如何导出内容，并留下本地创作痕迹。

Story Studio 不评价你的故事，也不替你做创作判断。它更像一张可编辑的导演工作台：把隐形的信息摆出来，让创作者自己决定下一步。

## 现在可以做什么

### 项目大厅

- 新建 Story Studio 项目。
- 打开已有项目。
- 查看最近使用的项目。
- 进入默认或上次使用的工作台。

### 七个工作台

| 工作台 | 当前定位 | 预览版能力 |
| --- | --- | --- |
| 灵感 | 创作输入层 | 用自由画布收集和组织灵感卡片、素材节点、连线与绘图关系。 |
| 世界 | 外围规则层 | 用对象、分类、颜色和画布关系整理世界观、地点、阵营、规则和道具。 |
| 人物 | 故事动力层 | 管理人物列表、人物维度、人物关系、卡司宇宙和问询入口。 |
| 结构 | 宏观组织层 | 按项目、幕、序列、场景组织故事，并通过自定义价值轴观察结构变化。 |
| 场景 | 实际创作层 | 在剧本式编辑器中写作，并结合结构栏、分页视图和右侧节拍板拆分场景推进。 |
| 提醒 | 创作约束层 | 用分类便利贴记录提醒，并投放到指定工作台、幕、序列或场景范围。 |
| 导出 | 输出与归档层 | 预览和导出 PDF，并生成/验证本地原创证明文件。 |

这些工作台已经有可运行的预览版界面，但仍处于快速迭代阶段。部分字段、交互和数据结构后续会继续调整。

### 场景写作

场景工作台是当前最核心的创作界面之一。它把正文编辑、结构位置和场景节拍放在同一个视野里：

- 使用 TipTap / ProseMirror 驱动的剧本式编辑器。
- 支持分页视图，用接近纸面排版的方式查看剧本内容。
- 左侧联动结构工作台中的幕、序列和场景。
- 上方可显示当前位置相关的价值曲线。
- 右侧节拍板用于把场景拆成更小的推进单元。

目标不是把故事强行套进模板，而是在写作时让创作者始终看见“我现在写的是整个故事里的哪一段”。

### 结构与价值曲线

结构工作台用于组织故事的宏观层级：项目、幕、序列、场景。创作者可以为不同层级设置价值轴，用曲线观察故事在不同维度上的变化。

这不是自动评分系统，也不是剧情建议系统。它只是把创作者自己设定的结构信息可视化，帮助回看节奏、对照设计和定位问题。

### 提醒系统

提醒工作台提供一组面向创作者的便利贴。提醒可以被分类，并投放到特定工作台或故事结构范围中。

它适合记录类似这样的约束：

- 某个人物在这一幕不能知道真相。
- 某个道具必须在后续场景回收。
- 某条情绪线不能在这个序列里提前释放。
- 某种叙事风格只适用于特定段落。

### 导出与本地原创证明

导出工作台目前包含 PDF 预览/导出入口，以及本地原创证明相关功能。

原创证明功能的定位是：为当前项目生成一份本地证明文件，用于记录项目内容摘要、文件信息和创作痕迹，并支持之后进行自洽验证。

它不是：

- 版权登记。
- 法律公证。
- 可信时间戳服务。
- 云端存证服务。

Story Studio 不会因为生成证明文件而替代正式的版权登记、法律咨询或第三方存证流程。

## Story Studio 不是什么

- **不是 AI 写作器**：它不以自动生成剧情、正文或评价故事为核心。
- **不是云端协作平台**：当前阶段优先保证本地桌面体验和本地项目文件可控。
- **不是模板化故事机器**：它提供结构工具和可视化反馈，但不强迫创作者套用固定公式。
- **不是稳定版生产软件**：当前仍是 alpha 预览版，项目文件格式和功能细节可能变化。

## 本地开发

### 安装依赖

```bash
npm install
```

### 启动开发模式

```bash
npm run dev
```

### 类型检查

```bash
npm run typecheck
```

### 生产构建

```bash
npm run build
```

### 预览构建产物

```bash
npm run start
```

```bash
# Windows
npm run build:win

# macOS
npm run build:mac

# Linux
npm run build:linux
```

## 参与与反馈

Story Studio 是一个个人独立开发并持续迭代的开源项目。

如果你关注长篇故事创作、剧本工程、游戏叙事、动画前期、拉片分析或创作者工具，欢迎试用预览版并反馈真实创作场景中的问题。

最有价值的反馈通常不是“加一个功能”，而是：

- 你在什么创作场景里卡住了。
- 现有工作台有没有帮助你看清问题。
- 哪个界面让你感觉打断创作。
- 哪些信息在长篇创作中最容易丢失。
- 哪些导出或归档能力会影响你是否敢正式使用。
